# Security Policy (보안 정책)

## Supported Versions (지원되는 버전)

The following versions of this project are currently being supported with security updates.
현재 보안 업데이트가 지원되는 프로젝트 버전입니다.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability (보안 취약점 보고 방법)

We take the security of this project seriously. If you believe you have found a security vulnerability, please report it using the following methods:
보안 취약점을 발견하셨다면 아래 절차에 따라 보고해 주세요.

1. **GitHub Private Vulnerability Reporting (Recommended)**: 
   We highly recommend using the 'Report a vulnerability' button under the **Security** tab of this repository. This allows for a secure, private discussion between the maintainer and the researcher.
   **비공개 제보 (권장)**: 리포지토리의 'Security' 탭에서 'Report a vulnerability' 버튼을 통해 비공개로 제보해 주세요. 제보자와 관리자 간의 안전한 소통이 가능합니다.

2. **Response Time (응답 시간)**: 
   You can expect an initial response within 48 hours of your report. We will keep you updated on the progress of the fix.
   제보 접수 후 48시간 이내에 확인 답신을 드리며, 해결 과정을 지속적으로 공유해 드립니다.

**Warning (주의):** Please do not disclose security vulnerabilities publicly via GitHub Issues. 
보안 결함을 GitHub Issue에 공개적으로 게시하지 마세요. 모든 사용자에게 위험을 초래할 수 있습니다.
